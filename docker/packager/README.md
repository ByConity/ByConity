# README

Allow to build ByConity in Docker for different platforms with different compilers and build settings. 
Correctly configured Docker daemon is single dependency.

Usage:
First step, at the root directory of the source code, rm folder `build_docker` if it exist

For example to build a deb package:

``` sh
mkdir -p deb/test_output/
mkdir -p /data01/$USER/.ccache_for_docker
./packager --output-dir deb/test_output/ --package-type deb --docker-image-version 0.1 --ccache_dir=/data01/$USER/.ccache_for_docker --version 0.1.1.1
ls -l deb/test_output/
```

$ mkdir deb/test_output/

``` sh
tests/ci/version_helper.py --update patch|minor|major 
```
