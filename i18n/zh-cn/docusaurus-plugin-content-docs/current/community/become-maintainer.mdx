---
title: 如何成为 ByConity 的维护人员和 TSC 成员
tags:
  - Docs
---

贡献像ByConity这样的开源项目可能会让人无所适从，但它也可以成为一次有价值的经历。成为维护者或技术指导委员会（TSC）成员是一个很大的成就，需要技术和社区贡献。以下是如何成为 ByConity 的维护者和 TSC 成员：

**步骤1：选择一个 issue **

* 第一步是找到一个要解决的 issue 或新功能。
* 您可以开始查找标记为“good first issue”的问题或加入现有的 PR 请求审查。
* 或者邀请维护者解释如何贡献，来获取相关的信息。

**步骤2：发起 PR 请求**

* 找到要解决的问题或功能后，您可以通过 fork 代码库并在本地上进行工作来开始。
* 当您准备好时，使用[约定提交规范](https://www.conventionalcommits.org/en/v1.0.0/#summary)创建一个适当标题的 PR 请求。
* 确保针对 GitHub 检查测试您的修改，以确保这些检查全部通过。

**步骤3：合并 PR 请求**

* 提交拉取请求后，请给维护者一些时间进行审查。
* 如果没有人注意到您的PR，请随时通过Slack或GitHub提醒维护者。
* 在提交之前确保所有PR检查都通过，以确保您的PR能够快速合并。

**步骤4：成为贡献者**

* 在为ByConity做出多次贡献后，您将被认为是贡献者，并且您将有权将代码提交到存储库中。
* 您可以利用这个机会审查其他开发者的 PR 请求并回答用户的问题，这将有助于您更熟悉代码。

**步骤5：成为维护者**

* 作为贡献者，您可以继续为项目做出贡献，并与其他贡献者分享您的知识，以获得足够的信用成为维护者。
* 成为维护者后，您将有更多的责任，如审查和合并 PR 请求、管理问题和确保代码质量。

**步骤6：成为 TSC 成员**

* 在成为维护者并展示技术和社区影响力后，您可以申请成为TSC成员。
* 作为 TSC 成员，您将有权对社区相关决策进行投票，并提名其他活跃的贡献者成为贡献者。
* 成为 TSC 成员是一项巨大的责任，它需要对项目及其社区的奉献精神。

请记住，贡献开源项目最重要的是要享受其中、学到新东西，并建立一个社区。祝好运！
