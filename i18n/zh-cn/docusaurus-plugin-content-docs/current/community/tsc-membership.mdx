# 加入ByConity技术指导委员会

本指南旨在提供如何成为ByConity技术指导委员会（TSC）成员的信息。它概述了ByConity中的开发角色，TSC的定义，成立的原因，基本原则，成为TSC成员的好处，以及成为TSC成员的步骤。

## ByConity的开发角色
* **用户：** 使用ByConity软件并通过提交错误报告和功能建议来提供反馈的个人。
* **贡献者：** 向ByConity计划贡献代码、文档、拉取请求评审、设计或其他技术成果的社区成员。
* **提交者：** 拥有修改（“提交”）项目存储库中的源代码、文档或其他技术成果能力的贡献者。
* **维护者：** 由于其在项目开发和演进方面的贡献而被选举的提交者。他们可以访问代码存储库，可以对社区相关决策进行投票，并可以提议其他活跃贡献者成为提交者。

## 什么是技术指导委员会？
TSC是一个由维护者组成的团队，负责维护和管理 ByConity 项目。它负责做出影响整个项目发展方向的决策，而不仅仅是一个代码仓。TSC成员承诺参与投票过程。

## 为什么成立TSC
当ByConity在2023年开源时，成立TSC是为了确保一个开放的治理体系，促进计划中所有参与者的所有权和中立性。

TSC成员的一些基本原则：
* 没有拥有特权的创始成员
* 所有贡献者平等
* 异步决策制定
* 每个维护者都有资格自动成为TSC成员

## 成为TSC成员的好处

作为TSC成员，您将享受以下好处:

* 获得ByConity社区的认可
* 参与涉及整个计划的议题的投票过程
* 参与计划的关键决策。

## 如何成为TSC成员

任何维护者都可以成为TSC成员。要成为TSC成员，您需要成为ByConity计划中的一个项目的维护者。

您可以通过在ByConity组织内执行以下至少一项操作来成为维护者：
* 支持和提交现有项目
* 被其他维护者邀请加入他们成为维护者。
